About

Apple Trailer Viewer is a HTML5/JS sample application that I built to teach myself how to use the latest and greatest tools for modern web application development.

View this project on GitHub

The original version of this app was created in 2010 in a day using Adobe Flex 4.1 as a request for a code sample for a UI Developer position, which, I think, sealed the deal (well, that, and my perspicacity and gregarious personality). However, now that "Flash isn't cool anymore" (except, maybe, in sweden), I've begrudgingly started playing with HTML5/JS so that I can be cool again, which is the most important thing to me IN THE WORLD.

Tools of the trade

What I used to build this app, and why

RequireJS

This handles the ordered and/or lazy loading of all the JS frameworks, libraries and files in the application, as well as the template snippets via the text plugin. The killer feature though, is that it requests dependencies ONLY when they are actually needed/used. Awesome!

jQuery

I used this for all the DOM manipulation and animation for the application. It's a hard requirement for using many of the other components and plugins in the app (particularly twitter bootstrap) but only a soft requirement for using Backbone.

Underscore

Utility library of JS functions which I've used primarily templating. It is the only hard requirement for backbone, but I would probably end up using it anyway because of it's usefulness.

Backbone

This is the glue that holds the app together. It facilitates a MVC (Model/Collections, View & Controller/Router) structure for the application which, when done correctly, allows for a separation of concerns approach therefore dramatically improves the maintainability of the app.

Twitter Bootstrap

The HTML5/CSS framework upon which the entire look and feel of the application is based. I used it because it provides an excellent base from which almost anything can built... and because while I primarily wanted to focus on learning the other tools rather than worrying too much about layout and div alignment, I still wanted it to look good. Done, and done.

Datejs

The name of this library is self-explanatory... I use it to format the dates that are displayed to the user.

What I've learnt

Some of the gotcha's, all of the lessons

Lesson 1.

I spent a lot of time trying to understand how RequireJS worked and got a little confused between difference of how version 1.0x works versus 2.0.x. While both have the ability to control the critically important order with which your JS is loaded, version 2.x uses a "shim" option in the configuration which makes it easier to control the inclusion and export of dependencies, and breaks the need for the order plugin which version 1.0.x used for that requirement.

Lesson 2.

Using templates for the application was worth the effort. I can make changes to very specific parts of the application without worrying too much breaking anything else.

In hindsight though, I now understand the purpose of using a logic-less templating engine like mustache, because it would ostensibly make the template more re-usable, and, quite frankly, cleaner.

Lesson 3.

This is kind of JS-101, but, you always have to be careful that you are referencing the proper scope. Here's a general example of how to maintain proper scope:

            var scope = function(){
                // Store a reference to the current scope
                var that = this;
                var trailerListView = new TrailerListView();
                trailerListView.on( 'viewTrailerInfo', function( trailerIndex ){
                    // Use the cached reference to this here, to keep scope
                    that.viewTrailerInfo( trailerIndex );
                });
            };

Lesson 4.

I had to do some formatting of the incoming data, which I decided to do in the initialize function of the model. There is probably a better place to do this, but it works. I also used Backbone's set and get to add or update the model attributes so that when the model is parsed to JSON, it would be as it should be.
